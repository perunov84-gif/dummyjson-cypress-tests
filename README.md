# DummyJSON API tests (QA test task)

Набор API-автотестов для публичного API [DummyJSON](https://dummyjson.com), реализованный на **JavaScript + Cypress**.

Проверяемый флоу:
- авторизация;
- получение данных авторизованного пользователя;
- операции с корзиной;
- негативные проверки.

Тестовый пользователь из документации DummyJSON: `emilys` / `emilyspass`.

## Инструменты в задании

- JavaScript (Node.js)
- Cypress
- `cy.request` для API-проверок
- `cy.visit` для визуального подтверждения работы с `https://dummyjson.com`

## Как запустить

Установка зависимостей:

```bash
npm install
```

Headless-запуск (основная команда для CI/проверки):

```bash
npm test
```

Визуальный запуск в браузере (видно переходы на страницы документации):

```bash
npm run test:headed
```

или напрямую:

```bash
npx cypress run --headed --spec cypress/e2e/dummyjson-api.cy.js
```

Интерактивный Cypress UI:

```bash
npm run test:open
```

Далее: `E2E Testing` → выбрать браузер → `dummyjson-api.cy.js`.

## Реализованные проверки

1. Успешная авторизация: `POST /auth/login`
2. Неуспешная авторизация (неверный пароль): `POST /auth/login`
3. Получение текущего пользователя с токеном: `GET /auth/me`
4. Получение текущего пользователя без токена: `GET /auth/me`
5. Получение корзин пользователя: `GET /carts/user/{userId}`
6. Получение корзины по id: `GET /carts/{cartId}`
7. Создание корзины: `POST /carts/add`
8. Обновление корзины: `PATCH /carts/{cartId}`
9. Удаление корзины: `DELETE /carts/{cartId}`
10. Негативная проверка корзины: `GET /carts/{несуществующий cartId}`

На каждый метод — 1–2 осмысленных assertion (статус-код + ключевые поля ответа).

## Примечания по поведению API

DummyJSON — демонстрационное API. Операции `POST /carts/add`, `PATCH/PUT /carts/{id}` и `DELETE /carts/{id}` **симулируют** изменения и не сохраняют их на сервере (согласно документации).

Страницы документации (`/docs/auth`, `/docs/carts`) могут загружаться медленно в Cypress, поэтому полный прогон с визуальными переходами занимает несколько минут. Это ожидаемо и не влияет на корректность API-проверок.
